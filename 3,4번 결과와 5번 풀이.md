# 5번 풀이
1. 더 작은 값을 가질 수 있다.
2. 항상 가장 작은 값을 가지지 않는다.
3. n/2개 이다.
4. 최악의 경우는 Θ(log n)이고, 최선의 경우는 Θ(1)이다.
5. 간단한 일이다.
6. Sift up 알고리즘은 시간 복잡도 측면에서 O(n\log n)이 소요되어 O(n)인 Sift down 알고리즘보다 느리다. 그러므로 데이터를 힙으로 변환할 때는 아래쪽에서부터 시작하는 buildHeap() 방식을 사용하는 것이 훨씬 유리하다.
7. 최대 힙에서 특정 원소의 값을 높였을 때, 해당 노드가 부모보다 커지면 힙의 불변성이 깨지게 되므로 Sift up 알고리즘을 사용해야한다. 값이 변경된 노드의 인덱스를 i라고 할 때, 해당 노드를 부모 노드와 비교한. 이때 부모 노드의 위치는 (i-1)/2로 계산할 수 있다. 만약 변경된 노드의 값이 부모 노드보다 크다면, 힙의 성질을 위반한 것이므로 두 노드의 위치를 맞바꾼다. 교환 후, 이동한 노드를 새로운 부모와 다시 비교한다. 이 과정을 노드가 루트에 도달하거나, 부모 노드의 값이 현재 노드보다 크거나 같아질 때까지 계속 진행하여 수선한다.
